Examen de Git Básico

Demuestra tus conocimientos de Git con este examen práctico.

Instrucciones Generales

Duración: 1 hora
Entrega: Documento Word (.docx)
Entorno: Solo terminal (sin GUI)
Puntaje total: 10 puntos
Incluir: Capturas + descripciones
Directorio: Limpio y nuevo

Cuestionario del Examen

1. Configuración de Git (1 punto)

Ajusta tu nombre de usuario y correo electrónico global de Git. Verifica que la configuración se haya aplicado correctamente.

2. Creación y clonación de repositorio remoto (1.5 puntos)

En GitHub, crea un repositorio llamado examen-<tu_nombre>. Cópia la URL del repositorio y clónalo en tu máquina local. Explica qué archivos y carpetas ocultas genera Git tras la clonación.

3. Área de staging y primer commit (2 puntos)

Crea un archivo hola.txt con el texto "Hola Git". Añade este archivo al área de staging y realiza el commit inicial. Muestra el historial de commits y comenta brevemente la estructura del log.

4. Trabajo con ramas (1.5 puntos)

Crea una rama nueva llamada feature y cámbiate a ella. Modifica hola.txt incorporando una línea adicional. Realiza un commit de estos cambios.

5. Merge de ramas (2 puntos)

Antes de fusionar, genera un conflicto: en la rama main, modifica la misma línea de hola.txt que editaste en feature y haz commit. Regresa a la rama main, fusiona la rama feature, y describe el conflicto que apareció explicando detalladamente cómo lo resolviste.

6. Push al repositorio remoto (2 puntos)

Sube los cambios de la rama principal (main) al servidor remoto. Verifica en la plataforma que tu historial de commits esté disponible.
